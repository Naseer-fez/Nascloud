# Architectural Rules

## Dual-Server Authentication Flow
- **SetupGUI**: A Python `.exe` GUI located in the `package` folder. It configures the environment and stores variables in `config.json`.
- **Config file**: `config.json` must be stored at the root of the codebase where both the GUI and the backend (`start.exe`) can access it easily.
- **Login Flow**:
  1. The Frontend first contacts the **Main Server** using the Central API Key.
  2. The user enters a frontend access code (and optional password).
  3. The Main Server verifies this and returns the **Client Backend Link** (e.g. an ngrok link generated by the client's `start.exe`).
  4. The Frontend dynamically updates its base URL (via React Context) to point to the Client Server for all subsequent requests.
  5. If `config.json` allows user registration, the Frontend displays a secondary "Dual Login Page" for the user to register/login directly with the Client Server.
